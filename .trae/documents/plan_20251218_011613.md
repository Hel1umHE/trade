# 管理后台设计与实现计划

## 1. 项目结构设计

* 创建 `admin` 文件夹，用于存放管理后台相关文件

* 创建 `admin/components` 文件夹，用于存放管理后台的组件

## 2. 页面设计

### 2.1 管理后台登录页面 (`admin/login.html`)

* 设计简洁的登录界面，只允许特定管理员账号登录

* 使用硬编码的管理员账号密码（如：admin/admin123）

* 登录成功后跳转到管理后台主页面

### 2.2 管理后台主页面 (`admin/index.html`)

* 采用与 `index.html` 一致的布局，包含侧边栏和主内容区

* 侧边栏包含四个板块：

  * 总览：显示平台统计数据

  * 商品管理：管理平台所有商品

  * 用户管理：管理平台所有用户

  * 订单管理：管理平台所有订单

* 主内容区使用 `router-view` 加载不同板块的内容

### 2.3 组件设计

创建以下组件文件：

* `admin/components/dashboard.js`：总览板块

* `admin/components/products.js`：商品管理板块

* `admin/components/users.js`：用户管理板块

* `admin/components/orders.js`：订单管理板块

## 3. 功能实现

### 3.1 登录功能

* 管理后台登录页面只允许特定管理员账号登录

* 登录状态保存在 `localStorage` 中

* 实现退出登录功能

### 3.2 总览板块

* 显示商品总数、用户总数、订单总数等统计数据

### 3.3 商品管理板块

* 显示所有商品列表

* 支持商品搜索、筛选、编辑和删除功能

### 3.4 用户管理板块

* 显示所有用户列表

* 支持用户搜索、筛选和编辑功能

### 3.5 订单管理板块

* 显示所有订单列表

* 支持订单搜索、筛选和状态更新功能

## 4. 集成现有系统

* 在 `account/login.html` 底部添加管理后台登录入口链接

* 确保管理后台与现有系统风格一致

## 5. 技术栈

* Vue 2.x

* Vue Router

* Bootstrap 5

* localStorage（用于数据存储和状态管理）
